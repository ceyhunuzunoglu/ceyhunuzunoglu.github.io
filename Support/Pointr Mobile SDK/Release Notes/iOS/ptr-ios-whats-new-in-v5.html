    iOS 5x Upgrade Guide

    Overview¶

    Here you’ll find detailed notes on the changes in the newest release of Pointr SDK for iOS

    SampleApp UI hierarchy change¶

    In the previous sample project, map was added directly to the main view controller and its view (self.view).

    In this release, we provide a multi-tab prototype, and to demonstrate the adding of the MapView to a frame smaller than full-screen, we created a containerView in Main.storyboard, where we add the MapView.

    In your project, you can either

    • Create ​your own containerView to contain the map or
    • Assign containerView = self.view ​ to make MapView fill the screen that is displayed

    Be careful

    If you miss this step and fail to add the MapView, A blank screen will show (as the MapView is not present in the View Hierarchy)

    Target version¶

    As of this version, Pointr SDK has removed support for iOS 7, and now minimally supports iOS 8.0 and up

    Previous New
    Min target version: 7.0 Min target version: 8.0

    Bitcode support¶

    This version does not support Bitcode generation.

    PointrMode¶

    PointrMode is changed to a Class.

    Previous New
    PointrModeProd [PointrReleaseMode mode]
    PointrModeDebug [PointrDebugMode mode]

    Joystick for testing¶

    Pointr iOS SDK now also supports use of a joystick to manually position yourself on the map. As with Android, this is intended to be a debug feature - not something that you will expose to your Users. To activate the Joystick, set mapView.shouldShowJoyStick to YES.

    To start off the mocked Joystick positioning, you need to hit one of the UP / DOWN / RR / RL buttons first.

    Info

    Note, on iOS when Joystick is enabled, unlike Android, Bluetooth reading is disabled to avoid conflicting the two positions.

    Previous New
    PTRMapView.shouldShowJoyStick

    Libraries to include in project¶

    You may refer to provided sample project also.

    • Accelerate.framework
    • AudioToolbox.framework
    • SystemConfiguration.framework
    • AVFoundation.framework
    • GLKit.framework
    • AVKit.framework
    • CoreBluetooth.framework
    • CoreLocation.framework
    • CoreMotion.framework
    • Pointr.framework
    • libsqlite3.tbd
    • libstdc++.tbd
    • libz.tbd
    • libcurl.a (can be found in Pointr.embededframework file)

    PTRMapView methods¶

    Previous New
    PTRMapView.startPathfinding Deprecated Use PathManager.startPathfinding instead

    PTRMapViewDataSource methods¶

    PTRAnnotationView is now deprecated. Rather than annotations, Pointr SDK now uses Drawable interface (as described above in the document). Drawables represent any item that could be drawn on the Map. Line drawing is also exposed to allow you full customisation of Path drawing.

    Previous New
    (PTRAnnotationView *)mapView:viewForAnnotation: (PTRDrawablePointView)mapView:viewForDrawable:
    (none) (CAShapeLayer) mapView:layerForRegionDrawable

    PTRMapViewDelegate methods¶

    Previous New
    (void)mapView:onPoiViewTouched: (void)mapView:onDrawablePointViewTouched:
    (void)mapView:onMapLevelChangedTo: Same
    (void)mapView:onMapRotationModeChangedTo: Same
    (void)mapView:didReceiveSingleFingerTap: Same
    (void)mapView:didZoomToLevel: Same
    (void)mapView:onPathfindingStarted: Deprecated: Use PathManagerDelegate callbacks instead
    (void)mapView:onUserCancelledPathfinding: Deprecated: Use PathManagerDelegate callbacks instead
    (void)mapView:onUserHasArrivedAtDestination: Deprecated: Use PathManagerDelegate callbacks instead
    (void)mapViewUserDidHideFooterView Deprecated: Footer View is now accessed via the Container class
    (PTRPathManagerSuggestionCallback)mapViewWillAskUserForSmartSuggestionCallback: Deprecated: This functionality is not supported in this version
    (none) (void)mapView:didInitializeSuccessfully:
    (none) (void)mapView:onLoadingStateChangedTo:
    (none) (void)mapView:onScreenRectChanged:

    PTRMapLoadingState¶

    PTRMapView now has a property to specify its current loading state, which can be any one of the following states:

    • PTRMapLoadingStateOff
    • PTRMapLoadingStateLoading
    • PTRMapLoadingStateReady
    • PTRMapLoadingStateError
    Previous New
    (none) PTRMapView.loadingState

    Further Development Notes¶

    Shrinking userPinView¶

    Shrinking userPinView for mapViews zoom level should be done in ContainerView if this is desired

    //can be used if smaller userpin when map zoomed out
- (void)mapView:(PTRMapView *)mapView didZoomToLevel:(float)zoomLevel {

    dispatch_async(dispatch_get_main_queue(), ^{
        //Change size of userpin in terms of zoom level
        [self shrinkUserPinView];
    });
}

    MapModes¶

    MapModes are now totally controlled by your in the ContainerBaseVC, so if the MapView level is changed by user (e.g. with a level selector view) and current position is on another level mode should be changed to free (otherwise map will zoom to user position for tracking mode). For example, the following should be in your MapView: onMapLevelChangedTo: delegate

    ...
if (level != _mapView.currentLocation.location.level) {
    dispatch_async(dispatch_get_main_queue(), ^{
        [_mapModeView setCurrentMapMode:PTRMapModeFree];
        [mapView setCurrentMapMode:PTRMapModeFree];
    });

}
...

    IKEA has a store layout added from PointrCloud and this prevents user to see blue empty areas in the screen to enable this feature in every level the following should also be added to your MapView: onMapLevelChangedTo: delegate

    dispatch_async(dispatch_get_main_queue(), ^{

//if there is a display area for this facility and level zoom to its center for not showing the other empty places

    id<PTRRegionDrawable> storeLayout = [Pointr.sharedInstance.getPoiManager

    getStoreLayoutPoiForLevel:mapView.currentLevel forFacility:mapView.currentFacility];

    if (storeLayout != nil && storeLayout.regionArray != nil) {
        CGRect layoutLimits = [self storeLayoutRectangleFromRegionArray:storeLayout.regionArray];
        mapView.layoutLimits = layoutLimits;
        CGFloat zoom = mapView.minimumZoomScale + 0.01;
        CGPoint zoomPosition = storeLayout.centerPoint;
        [mapView zoomViewToX:zoomPosition.x andY:zoomPosition.y withZoomLevel:zoom];
    }
    [self drawPoisInMapView:mapView];
});

    Info

    Zooming logic is added in the above to be sure the Map is not below the update minimum

    (void)mapView:(PTRMapView *)mapView onPoiViewTouched:(PTRPoi *)

    method has been changed to

    (void)mapView:(PTRMapView *)mapView onDrawablePointViewTouched:(PTRDrawablePointView *)drawablePointView

    Poi selection and view enlarging from tap will be made in container from now on please check inside new method mapView:onDrawablePointViewTouched:

    When map mode is changed mapView:onMapRotationModeChangedTo: callback is fired and from now on every change due to mode change will be made on container please check doModeChangeEventsWithMapMode:(PTRMapMode)mapMode function inside the mapView:onMapRotationModeChangedTo: delegate

    mapView:didReceiveSingleFingerTap: method is fired when a touch on mapView happened (if a touch on drawablePointView happens this callback is not fired) deselecting drawablePointView and containerState change should be done in this callback (see deselectDrawablePointView: and handleSingleTapOnMapview: methods in didReceiveSingleFingerTap:)

    The new callback mapView:didInitializeSuccessfully: has been added to MapView and can be used to orchestrate events that should be carried out directly after MapView initialization

    The new callback mapView:onScreenRectChanged: is added to mapView to redraw pois on MapView.

    Info

    The Screen Rect can also be changed at initialization of the MapView, and on level changes if mapSizes are not same at levels

    Delegates¶

    PTRPoiManagerDelegate Methods¶

    PoiManager does nothing by itself from now on these should be done in Container using delegate methods

    onPoiManagerSelectedPoi: method can be used for enlarging and shrinking views

    onPoiManagerCurrentPoisUpdated: method should be used for re drawing pois in MapView

    onPoiManagerEnterTrigger: method can be used for trigger handling

    PTRSearchViewDelegate Method¶

    searchViewStartedPathFinding gets called when start call comes from searchView no method

    [self.mapView startPathfinding] can be used for starting the Pathfinding experience, however you should use [[[Pointr sharedInstance] getPathManager startPathfinding];

    PTRPositionManagerDelegate Methods¶

    onPositionManagerPositionIsFading and onPositionManagerPositionIsLost should be used if any change in view will be done

    onPositionManagerCalculatedLocation: is a replace for onPositionManagerCalculatedPosition: for MapView to respond those position changes this calculated location should be passed to mapView and do events with this position (please check updateMapView: basedOnCalculatedLocation: function in container)

    onPositionManagerDetectedPositionLevelChange: should be used if level change will be made if positions level change

    PTRPathManagerDelegate Methods¶

    onPathManagerPathCalculationAborted: will be used for events to do after path is aborted

    onPathManagerHasArrivedAtDestination: can be used for events to do after path is completed too (eg. show a arrival alert)

    onPathManagerHasCalculatedPath: should pass path to mapView (please see changePathWithPath:inMapView: method in Container. All turning and destination icons must be added from the Container

    onPathManagerPathCalculationFailedWithErrorMessage: should be used for resetting path in mapView
