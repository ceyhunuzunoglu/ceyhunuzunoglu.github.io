    Quickstart - Pointr Cloud

    Get Started With Pointr Cloud¶

    Here you can find information about logging in, data analytics functions, map management, notification management, and settings and configurations, and other useful information.

    Disclaimer

    All maps and data shown in this document are for demonstration purposes only and not based on factual data. The images shown may differ in the document and on your screen depending on the user permissions and updates in Pointr Cloud.

    Email support in order to give feedback about this document.

    Basics¶

    These are the basic features to get you started with Pointr Cloud. The current version of Pointr Cloud may be seen on the left menu bar after logging in.

    Login¶

    In order to use the available features of Pointr Cloud, go to the web address that is provided. This address will be of the form xxx.pointr.cloud or xxx.pointrlabs.comwhere you should replacexxxwith your unique domain name.

    Note that in special cases, your Pointr Cloud endpoint will differ, but you should already know this - if you’re unsure, get in touch

    Enter your login details into the form, and click login. For the login screen see the image below. The welcome email you receive when you first register will contain username information. If you forget this, you can always refer to the email.

    Image 1.1: The login screen.

    Request Access¶

    If you are not registered, or you have not received a welcome email, you may click on the “request access” link which is located below the “Log in” button shown in Image 1.2.

    Image 1.2: Login details. Clicking the request access button will trigger an email to your administrator.

    1.1.2 Can’t remember your password? If you are given account details but cannot remember your password, you may click on the “password” link shown in Image 1.2, to request a new password.

    If a password request is attempted with an email address that is not a registered user, no action will be taken for security reasons. In case of too many attempts as a non- registered user, you will not be allowed access from the same device for a limited time.

    1.2 Venue Selection You can select the venue that you wish to work on from the drop-down menu on the left menu bar, next to your company logo (See Image 1.3).

    Image 1.3: Venues. If a venue or a store does not show in the dropdown menu, this might be because: You might not have the privileges to see all the venues of your company. The venue has not been added on to the system. In order to be able to see a venue on the drop-down menu, contact support. Analyse These are the tools that enable you to analyse the data collected from users.

    2.1 Analytics These charts show data gathered from wifi - i.e. POP devices. For charts that show analytics gathered from bluetooth data - i.e from beacons - see SDK Analytics screen.

    Image 2.4: Analytics charts. On this screen you have the option to visualise the data according to

    Various venues, Various user segments, Daily or hourly. Also on the upper right corner of the charts, you have the option to export the charts as .xls or .pdf files.

    Pointr Cloud Tutorial v3. If there are multiples venues on the Pointr Cloud to which you have access, you will also see these in tabs above each chart. You can select these tabs to view the charts for each venue.

    Note that the charts on this module are customisable. Please contact support to discuss if you wish to customise the analytics charts.

    2.2 Zone Analytics This chart allows you to visualise the zone-wise data. To be able to see the analytics on this module, it is necessary to define “Zone” or “Customer Zone” as a POI at Map Designer.

    Image 2.5: Zone Analytics screen. You can select a start and end date that you wish to visualise the data for. You can see the date and time selection boxes on top of the screen on Image 2.5. Next to the date selection boxes you can choose either Footfall to view the number of users or Dwell Time to view the time spent. Underneath the date selection (or next to it - depending on your screen resolution), you will see a dropdown menu to select “count” or “percentage”. Here, you can also choose to visualise the data according to count at each zone or the percentage of the count compared to the rest of the same level. Permanent Tooltip is for keeping the numerical data on top of the map permanently. When you are ready, press the “Run” button to see the data appearing on the map. You can refer to the legend on the bottom-right corner of the map for colour-coding. If you have not selected

    Pointr Cloud Tutorial v3. permanent tooltip, you can alternatively see the values by clicking on each zone as well.

    2.3 Customer Flow This tool allows you to visualise the customer flow (or flux) data using colour-coded arrowheads. To be able to see the analytics on this module, it is necessary to define “Zone” or “Customer Zone” as a POI at Map Designer. It shows the count or percentage of users passing from one neighbouring zone to the other along with the direction of passage.

    Image 2.6: Customer Flow chart. The options on this screen are the following.

    You can select the date-time interval on the top of the screen. (See Image 2.6) You can choose to see a count of customers or percentage compared to the whole of the level on the dropdown menu next to date selection. When you are ready, you can press “Start” button to see the data on the map. You can refer to the legend on the bottom-right corner of the map for colour-coding. You can also hover on the arrowheads to see the numerical values for the transition between the two zones.

    2.4 SDK Analytics In this section you can see an analysis of the data collected from SDK (software development kit)

    Pointr Cloud Tutorial v3. by using four different graphs. You can view the graphs week by week, by choosing the week from the dropdown menu on each graph. These graphs are explained below.

    Number of Sessions shows the total number of sessions made using SDK in the chosen time period. For example one user uses SDK three times and another uses it four times. The total number of sessions of these two users is seven. Number of Calculated Paths shows the number times SDK users made a search and used the calculated path. Number of Searches shows the total number of searches done in the chosen time period including the paths not taken. Number of Unique Users shows the number of users excluding the revisits. For example, one user uses SDK three times and another uses it four times. The number of unique users is two here. Note that if a user uses the SDK and deletes the app, then they download it and use it again, they will show as another unique user as they are given a new ID every time they download the app. 2.5 Heatmap This feature enables the cloud users to produce heatmaps using the data acquired from visitors. You can manage which information the map will show by using the dropdown options underlined in blue in Image 2.7. Here you can also specify the date and time interval.

    Image 2.7: Heatmap data. The options you can choose in order of appearance in Image 2.7 is as follows.

    Pointr Cloud Tutorial v3. Floor selection: Select which floor information you want to see. You may also choose to see all the floors next to each other by selecting “All floors”. Type of data: You may either viewdwell timeorvisitor count. When you choose dwell time, the colours will represent the time spent in one region from red (the most) to green (the least). If you choose visitor count, then the colours will show, most number of visits (red) to least number of visits (green). Source: This option, by default, has value “All”. Here you can choose the source that the data comes from. This can be SDK, bluetooth, wi-fi, bluetooth and wifi, asset wi-fi, asset bluetooth, all of them, unknown, and error. The data is acquired from POP devices instead of beacons if the source is not SDK. Error shows data identified as error. And unknown shows data that could not be identified. Date: Here you can choose the date and time interval that you wish to see the data in. Basic/Advanced: When this button shows “Advanced”, you are on the advanced screen where you choose a start and end date for the data. If the button shows “Basic”, the time interval is one day by default and you do not need to set a start date. Note that this button is planned for deprecation. 2.6 Zone Heatmap In this module you can view heatmaps of the customer zones that you added on Map Designer as a POI layer, at least 24 hours before using this analytics tool. You can view the zone heatmaps in 24-hour intervals.

    Image 2.8: Zone heatmap screen. Pointr Cloud Tutorial v3. Select beginning and end date and time. The interval has to be a maximum of 24 hours. Select either Footfall to view the number of users or Dwell Time to view the time spent. Select “Permanent Tooltip” in order to view the count and percentage in a speech box on the map. When you click “Run”, you will see colour-coded map like that in Image 2.8.

    2.7 Tracking Tracking allows for demonstration of active users’ locations as pin points. From the dropdown

    Image 2.9: Tracking screen. menus above the map, you can choose the following options.

    Floor Selection: Select which floor you want to see the data from. Source Selection: Select the source. The options are SDK, bluetooth, wi-fi, bluetooth and wi-fi, asset wi-fi, asset bluetooth, all, unknown, or error. Time Interval: Select the time interval you wish to see the data for, between now and a past time. This could be in minutes, hours, days, weeks, or months. 2.8 Simulation Pointr Cloud Tutorial v3. Simulation feature allows you to animate the tracking data collected from the visitors. Note that this is not a simulation of predicted future data, rather this is an animation of the already collected data. You can choose the data you want to simulate from the menu bar above the map. The

    Image 2.10: Simulation screen. options, in order of appearance, are the following:

    User selection: You may either simulate data from one anonymous user, or you may select the “All Users” option to view all the users. Source selection: It is also possible to select the source that the data was taken by. This can be SDK, bluetooth, wi-fi, bluetooth and wi-fi, asset wi-fi, asset bluetooth, all, unknown, or error. Time interval: You may choose the time interval you wish to simulate. This function cannot show a duration of time of more than 6 hours. If you choose a duration more than six hours, you will get an error message saying “Please select a time interval shorter than 6 hours”. Show Trace: If this option is selected, the simulation will show all the routes that the users took. If unselected, the simulation will only show the starting and ending points. Colorized: If selected, the simulation will show each user with a different colour. If unse- lected, all users will be shown with a single colour. Speed: In this dropdown menu, the speed of the simulation may be selected. The options are 5x, 1x, 2x, 10x, 20x, 50x, fastest. For example 5x means the simulation will be 5 times faster than real time. The fastest option shows the simulation as fast as it can be.     Frequently Asked Questions 1. I did not get an email when I used the “Can’t remember password” option. What should I do?

    This could be because either one of these causes:

    You might be at an incorrect domain address. Your email might not be registered. Contact your admin to resolve this issue.

    1. Why can I not see a venue that I want to work on?

    If you cannot see a venue in the dropdown menu, this might be because your account has not been given access to it. Contact your admin to request access to the venue.

    1. I cannot see any data points on the graphs? - Make sure that you selected the correct time interval where some data should have been collected. - Make sure that the store is selected. - Check if the beacons or POP devices are working from Settings and Configurations.
    2. Heatmap/Simulation show visitor data on the map where they cannot go (e.g staff only areas, walls.)

    Make sure that the beacon or POP device coordinates are set up correctly on map designer if this is applicable.

    1. I added a map and saved it but it is not visible to app users.

    Make sure to publish the map after saving. Users can only see the map if it is published after saving.

    1. I want to add a POI layer type that is not one of the options. What can I do?

    Make sure that the POI type you wish to add cannot be added as one of the options in the drop- down list. Contact your admin.

    1. I select POI layer and click the green plus sign, nothing happens. What should I do?

    You should select one of the POI type options that is below “POI layer” in the dropdown menu. “POI layer” itself is not a POI layer type.
